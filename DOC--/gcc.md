## 函数声明与定义/实现

声明：int add (int a,int b);

定义/实现:
int add (int a,int b)
{a+b}

## 下面以动态链接为例，讲述链接与执行的差异

### 链接
我们说使用某个库，首先引入其头文件，头文件就可以看作是一种函数的声明

编译得到的.o二进制文件有一张符号表，表明哪些函数没有被定义/实现，

把符号表理解为两列，左边是函数声明，右边是函数实现：
<img width="162" height="86" alt="image" src="https://github.com/user-attachments/assets/446197ef-3092-4c51-ad91-aae519a7ef78" />


然后进入链接阶段，

然后链接器对于没有定义/实现的函数，开始去库里面寻找，当然，库里面都是二进制，.so之类的

读取.so文件，可以看到.so文件实现的是函数列表

注意，找到.so后，并不会存储.so文件的具体路径以及.so文件的子依赖，然后链接完成得到的二进制只是又声明 NEED .so之类的


### 执行
执行二进制，要处理二进制声明的 NEED .so，即继续寻找，然后加载到内存里面，以及还会递归查询.so的子依赖，.so,.so等等

说明执行的需要的.so远比编译阶段进行检查的多的多
